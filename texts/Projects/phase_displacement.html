Phase displacement measurements¶

Instantaneous magnitude difference¶

If two signals have equal amplitude the displacement can be measured as the difference in magnitude between samples of each waveforms taken at the same instant. E.g. by subtracting one sine wave from the other, a new signal will emerge, with zero amplitude for zero phase displacement.

Discrete fourier transform (DFT)¶

If you know the frequency of the signal that you would like to measure, a single frequency DFT can be used to detect the phase difference.

For each of the signals the following equations will provide the phase relative to a reference:

\[x_i = \sum_{n = 0}^{N - 1} \cos(f \cdot t_n) \cdot x_n\]
\[x_q = \sum_{n = 0}^{N - 1} \sin(f \cdot t_n) \cdot x_n\]
\[\theta = \textnormal{arctan2}(x_i, x_q)\]

If both signals have the same reference, the phase difference can be computed by:

\[\varphi = \theta_1 - \theta_2\]

The reference for each signal will be the instant we start the sampling. This is not important however, since we are only looking at the difference. In order to avoid confusion it can be advantageous to normalize the angle \(\varphi\) from -180 to 180 degrees. Otherwise the computation might sometimes produce results such as -340, instead of +20. An approximately steady phase displacement should not have such large swings in numerical value, even though the value represents the same angle.

The following code listing illustrates an implementation for the sampling and calculation of the phase displacement between two 50 Hz waveforms.

  4 * @brief DFT for phase difference estimation 
  5 * @version 0.1

  8 * @copyright Copyright (c) 2022
  9 *
 10 * Single frequency DFT is easy to implement, and suitable for estimation of phase difference between
 11 * two sine waves at the same frequency, as long as the frequency is known. If the frequency is slightly
 12 * different, as in a synchronoscope where we want to connect two generators which generate slightly
 13 * different frequencies, another approach (possibly a PLL?) is needed.
 14 * 
 15 * This version of the software uses floating point, hence it is not too effective on the Atmega328p.
 16 * 
 17 * This implementation takes a given number of samples, and then compute the DFT. Alternatively it
 18 * could have been implemented to continuosly compute the DFT as each new sample comes in.

 20 */

 92float compute_rms(const float x[], const size_t N){
 93
 94  float x_rms = 0;
 95
 96  for(uint8_t n = 0; n < N; n++){
 97    x_rms += x[n]*x[n];
 98  }
 99
100  x_rms = sqrt(x_rms/N);
101
102  return x_rms;
103}

191      float phase_diff = signal1_dft.phase - signal2_dft.phase;
192      float phase_diff_deg = phase_diff * (180 / PI);
193
194      phase_diff_deg = fmod(phase_diff_deg + 180, 360); // % does not work for floating point numbers, hence we use fmod().
195      if(phase_diff_deg < 0)
196        phase_diff_deg += 360;
197
198      phase_diff_deg -= 180;
199
200      Serial.print("Phase: ");
201      Serial.println(phase_diff_deg);
202    }
203
204    sample_counter = 0;
205    delay(2000);
